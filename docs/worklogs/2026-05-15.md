# 工作日志：2026-05-15

## 操作概述

修复 MinerUService 输出收集逻辑，打通 MinerU 全管道解析 + FastAPI 端到端链路。

## 关键操作

1. **修复 `_collect_output`**：
   - MinerU 3.x 输出结构为 `{book_stem}/{pdf}.pdf/auto/{pdf}.md`
   - 改用 `rglob` 递归搜索，不需要硬编码路径
   - 增加 `MINERU_TYPE_MAP` 处理 MinerU 实际 block types

2. **修复 `_extract_pages`**：
   - 实际格式是 `list[list[dict]]`（索引即页码），非假设的 `list[dict]`
   - 增加 `_extract_text()` 处理嵌套 content dict
   - 增加 `_extract_latex()` 从多种 key 中提取公式
   - 增加 `_bbox_to_dict()` 将 list bbox 转为 dict

3. **移除 ThreadPoolExecutor**（Windows 多进程兼容性）：
   - MinerU 内部使用 `ProcessPoolExecutor`，嵌套在 ThreadPoolExecutor 中无法创建子进程
   - 改为直接同步调用

4. **添加环境变量注入**（`app/main.py`）：
   - `HF_ENDPOINT=https://hf-mirror.com`
   - `MINERU_MODEL_SOURCE=huggingface`
   - `HF_HUB_DISABLE_SYMLINKS_WARNING=1`
